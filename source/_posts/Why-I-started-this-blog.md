# Why I started this blog

I compared *VuePress* and *Hexo* before get started, as I have around 3 years of experiences on Vue.js, getting familiar with *VuePress* won't be very hard. Plus, I might take advantage of *Vue.js*, make a theme with Vue.js components, enhance its functionality. But after some research, I notices that *VuePress* doesn't have an official site for themes, unlike *Hexo* ([[https://hexo.io/themes/]](https://hexo.io/themes/%5D)(the theme site)). That means I have to search around GitHubs and NPM packages for a suitable theme, and this definitely takes more efforts than just go to the official website and download one. Plus, I should focus on writing things rather than building the blog itself. So at the end I choosed to use *Hexo* instead of *VuePress*.

I've already have a *Wordpress* blog hosted on a rented machine. I even bought a domain for it. After months for uses I found that it doesn't very suitable for a programmer's tech blog. It has very poor support for `codes` and formula highlighting (both inline and block codes). The limited colouring theme the highlighting plugin provides doesn't fit well with the blog's theme, and among all problems the worst is that it cannot display the highlighted codes in real-time.

That is the reason why I choosed to rebuild my blog with Hexo. It uses MarkDown, which has well support of code snippets. There are couples of great MarkDown editor I know, e.g. Typora, MarkText, even VSCode, which all provides well real-time preview, making me write article easier.

Of course there is one major drawback of Hexo, as so do other static website generators. That is it doesn't have a backend server. Without a backend server I cannot host demos that require a database or Node.js server. But hay, that is just overthinking - at least for now. Because currently I don't have any projects that needs a backend server. (as a front-end developer).
